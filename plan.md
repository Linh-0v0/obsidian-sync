# Obsidian Vault Two-Way Sync with Google Drive

## Overview

Use **rclone** to sync your Obsidian vault with Google Drive on a 30-minute interval via a systemd timer. No file watcher needed — rclone diffs the full vault each run, handling initial sync, offline catch-up, and batching automatically.

---

## How It Works

```
Every 30 minutes (systemd timer)
      |
      v
rclone bisync (two-way sync)
      |
      v
Compare local vault <-> Drive folder
      |
      +-- New/modified locally   -> upload to Drive
      +-- New/modified on Drive  -> download to local
      +-- Deleted locally        -> kept on Drive (no deletes)
      +-- Deleted on Drive       -> kept locally (no deletes)
      |
      v
Log result, retry failures
```

---

## File Structure

```
obsidian-sync/
├── sync.sh           # Main sync script (runs rclone + logging + retries)
├── rclone.conf       # rclone remote config (auto-generated by rclone setup)
├── sync.log          # Rotating log file
├── .env              # Vault path + Drive folder name
└── systemd/
    ├── obsidian-sync.service   # Systemd unit (runs sync.sh)
    └── obsidian-sync.timer     # Systemd timer (every 30 min)
```

---

## Key Decisions

| Decision | Choice | Reason |
|---|---|---|
| Sync engine | `rclone bisync` | Handles two-way sync, diffing, conflict detection out of the box |
| Interval | 30 minutes | Balances freshness vs. API quota; user preference |
| Deletes | Never propagate | `--no-delete` flag — Drive acts as append-only archive |
| Conflicts | Keep both copies | rclone renames the conflicting file with a `.conflict` suffix |
| Auth | rclone OAuth flow | Built-in browser-based auth, stores token in rclone.conf |

---

## Sync Script (`sync.sh`)

The script will:

1. Load `.env` for vault path and Drive folder name
2. Run `rclone bisync` with flags:
   - `--no-delete` — never delete on either side
   - `--conflict-resolve newer` — prefer the newer file on conflict
   - `--resilient` — don't abort on transient errors
   - `--recover` — recover from interrupted syncs
   - `--log-file sync.log` — append logs
   - `--log-level INFO`
3. On failure (non-zero exit):
   - Log the error
   - Retry up to 3 times with 60-second backoff
   - If all retries fail, write to a `sync-errors.log` for manual review
4. On success, log timestamp and file count

---

## Error Recovery

- **Network failure mid-sync**: rclone's `--resilient` and `--recover` flags handle interrupted transfers. Next run picks up where it left off.
- **Repeated failures**: After 3 retries, errors are logged to `sync-errors.log`. The timer still fires every 30 minutes, so the next cycle will retry automatically.
- **Conflict (same file edited on both devices)**: rclone keeps both copies — one gets a `.conflict` suffix so nothing is lost.

## Setup Steps (One-time)

1. **Install rclone**
   ```bash
   sudo apt install rclone
   # or: curl https://rclone.org/install.sh | sudo bash
   ```

2. **Configure Google Drive remote**
   ```bash
   rclone config
   # Choose: New remote -> name it "gdrive" -> Google Drive -> follow OAuth prompts
   ```

3. **Test the connection**
   ```bash
   rclone lsd gdrive:
   ```

4. **First sync (initialize bisync)**
   ```bash
   rclone bisync /path/to/vault gdrive:ObsidianVault --resync
   ```
   `--resync` is required on the very first run to establish the baseline.

5. **Install systemd timer**
   ```bash
   cp systemd/obsidian-sync.service ~/.config/systemd/user/
   cp systemd/obsidian-sync.timer ~/.config/systemd/user/
   systemctl --user enable --now obsidian-sync.timer
   ```

6. **Verify it's running**
   ```bash
   systemctl --user status obsidian-sync.timer
   journalctl --user -u obsidian-sync.service -f
   ```

## Why rclone Over Raw Google Drive API

- **Two-way sync built in** — no need to write diffing/conflict logic
- **No deletes handled natively** — single flag
- **Error recovery built in** — resilient mode, resume support
- **No Node.js dependency** — single binary, lower resource usage
- **Battle-tested** — widely used for cloud sync on Linux
- **Initial/offline sync is free** — every run is a full diff, no missed changes
